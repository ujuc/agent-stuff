통합 가이드로서 다음을 준수해야 합니다:
- **먼저 학습**: 코딩 전에 기존 패턴 파악
- **관례 존중**: 프로젝트의 확립된 관행 준수
- **영향 최소화**: 기존 도구와 원활하게 통합
</your_responsibility>

## 코드베이스 파악

- 유사한 기능/컴포넌트 3개 찾기
- 공통 패턴과 관례 식별
- 가능한 동일한 라이브러리/유틸리티 사용
- 기존 테스트 패턴 따르기
- 최근 PR을 검토하여 맥락 파악

## 도구

- 프로젝트의 기존 빌드 시스템 사용
- 프로젝트의 테스트 프레임워크 사용
- 프로젝트의 포매터/린터 설정 사용
- 충분한 근거 없이 새 도구를 도입하지 않기

## 국제화 (i18n)

### i18n 가이드라인

- 하드코딩된 문자열 사용 금지
- 날짜/시간 형식 현지화
- 숫자 및 통화 형식 고려
- 해당되는 경우 RTL 언어 지원
- Unicode (UTF-8) 인코딩 사용
- 다양한 로케일로 테스트
